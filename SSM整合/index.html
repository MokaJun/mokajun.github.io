SSM框架整合

Spring + Spring MVC + MyBatis

Spring MVC 负责实现 MVC 设计模式，MyBatis 负责数据持久层，Spring 负责管理 Spring MVC 和 MyBatis 相关对象的创建和依赖注入。

  • 创建 Maven 工程，pom.xml

  • web.xml 中启动Spring、 SpringMVC（其实就是前置servlet，字符编码filter、加载静态资源等Client-Sever的东西）。

    web.xml 是web项目的入口，所以要在这里启动好SpringMVC、Spring、字符编码过滤器、加载静态资源

    而且SpringMVC会把静态资源全部拦截，所以也需要在这里额外配置

    只要web项目一启动，就会到这里配置的classpath: spring.xml这里开始读取配置文件

  • 在 spring.xml 中配置 MyBatis 和 Spring 的整合。

    MyBatis主要靠SqlSessionFactory接口的实体类来运作，原生MyBatis是builder.build(config.xml)来生成defaultFactory

    但是用整合版的SqlSessionFactoryBean，很多参数(连接池+mapper地址)就直接作为属性写入了，只有少部分继续依靠读取config.xml

  • config.xml 配置一些 MyBatis 辅助信息，比如打印 SQL 等。
  • 配置 springmvc.xml
