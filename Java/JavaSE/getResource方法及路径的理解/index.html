getResource()方法及路径的理解

用JAVA获取文件，听似简单，用JAVA的File类，如要取得c:/test.txt文件，就会这样用File file = new File(“c:/test.txt”);

这样用有什么问题，相信大家都知道，就是路径硬编码，对于JAVA精神来说，应用应该一次成型，到处可用，并且从现实应用来讲，最终生成的应用也会部署到Windows外的操作系统中，对于linux来说，在应用中用了c:/这样的字样，就是失败.

所以，我们应该尽量避免使用硬编码，不要直接使用绝对路径。

比如我们有以下目录

|–project

​ |–src

​ |–com.southwind.Test

​ |–Test.java

​ |–file1.txt

​ |–file2.txt

​ |–classes

​ |–com.southwind.Test

​ |–Test.class

​ |–file3.txt

​ |–file4.txt

在上面的目录中，有一个src目录，这是JAVA源文件的目录，有一个classes目录，这是JAVA编译后文件(.class文件等）的存放目录

那么，我们在Test类中应该如何分别获得

file1.txt file2.txt file3.txt file4.txt这四个文件呢？

首先讲file3.txt与file4.txt

file3.txt:

方法一：File file3 = new File(Test.class.getResource(“file3.txt”).getFile());

方法二：File file3 = new File(Test.class.getResource(“/com.southwind.Test/file3.txt”).getFile());

方法三：File file3 = new File(Test.class.getClassLoader().getResource(“com.southwind.Test/file3.txt”).getFile());

file4.txt:

方法一：File file4 = new File(Test.class.getResource(“/file4.txt”).getFile());

方法二：File file4 = new File(Test.class.getClassLoader().getResource(“file4.txt”).getFile());

很好，我们可以有多种方法选择，但是file1与file2文件呢？如何获得？

答案是，你只能写上它们的绝对路径，不能像file3与file4一样用class.getResource()这种方法获得

或者在pom中加上build标签

关于原理

Class类的getResource(String name)方法

  • 会先调用Class类里的resolve(String name), 根据是否以”/“开头来重新返回一个解析后的reName
      - "/"开头的，去掉”/“，返回剩余的
  - 不以"/"开头的，返回类所在的路径
  • 再将解析后的reName作为参数传递到clazz.getClassLoader().getRecource(reName)里面去生成URL

ClassLoader类的getResource(String name)方法

直接以根目录为起点解析，返回URL

他们的本质都是调用ClassLoader的getRecource(String name)，以根目录为起点，只是clazz对象会把name根据”/“来解析成根目录为起点的String。

public java.net.URL getResource(String name) {
        name = resolveName(name);
        ClassLoader cl = getClassLoader0();
			//省略
        return cl.getResource(name);
    }


 private String resolveName(String name) {
        if (name == null) {
            return name;
        }
        if (!name.startsWith("/")) {  //对于不以/开头的文件，
            Class<?> c = this;   //获取当前加载类的完整的类路径，我这里是com.southwind.Test
            while (c.isArray()) {
                c = c.getComponentType();
            }
            String baseName = c.getName();
            int index = baseName.lastIndexOf('.');//找到类的目录名称
            if (index != -1) {
                name = baseName.substring(0, index).replace('.', '/')
                    +"/"+name;//将对应类的目录名称中的.替换为/ 并在最后加上/ 文件名
            }
        } else {
            name = name.substring(1);  //对于/开头的文件名，会只保留文件名称部分。
        }
        return name;
    }
System.out.println(Test.class.getResource("/"));
//CLass调用的方法，先clazz.resolve方法解析，以"/"开头，去掉"/"，返回根目录
//ClassLoader.getResource解析"根目录"，返回URL
//file:/E:/Note/Code/MyBatisNange/target/classes/
System.out.println(Test.class.getResource(""));
//CLass调用的方法，先clazz.resolve方法解析，不以"/"开头，返回的是类文件所在的文件夹
//ClassLoader.getResource解析"类所在目录"，返回URL
//file:/E:/Note/Code/MyBatisNange/target/classes/com/southwind/Test/


System.out.println(Test.class.getClassLoader().getResource("/"));
//ClassLoader.getResource解析 " 根目录 + 多余的/ " 返回URL
//根目录后有了多余的/ 解析不到文件
System.out.println(Test.class.getClassLoader().getResource(""));
//ClassLoader.getResource直接解析"根目录"，返回URL
//file:/E:/Note/Code/MyBatisNange/target/classes/

注：还有一个getResourceAsStream()方法，参数是与getResouce()方法是一样的，它相当于你用getResource()取得File文件后，再new InputStream(file)一样的结果
