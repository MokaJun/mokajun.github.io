  1. wait()方法必须放在while(判断是否wait)的代码块中，避免虚假唤醒

重入锁

ReentrantLock 限时性：判断某个线程在一定的时间内能否获取锁，通过 tryLock 方法来实现

boolean tryLock(long time,TimeUnit unit)

time 指时间数值

unit 时间单位

package com.southwind.demo;

import java.util.concurrent.TimeUnit;
import java.util.concurrent.locks.ReentrantLock;

public class Test {
    public static void main(String[] args) {
        TimeLock timeLock = new TimeLock();
        /**
         * A 拿到锁，执行业务代码，休眠 5 秒钟
         * B 尝试拿锁，需要在 3 秒钟之内拿到锁
         */
        new Thread(()->{
            timeLock.lock();
        },"A").start();
        new Thread(()->{
            timeLock.lock();
        },"B").start();
    }
}

class TimeLock{

    private ReentrantLock reentrantLock = new ReentrantLock();

    public void lock(){
        /**
         * 尝试在3S内获取锁
         */
        try {
            if(reentrantLock.tryLock(3, TimeUnit.SECONDS)){
                System.out.println(Thread.currentThread().getName()+" get lock");
                TimeUnit.SECONDS.sleep(5);
            }else{
                System.out.println(Thread.currentThread().getName()+" not lock");
            }
        } catch (InterruptedException e) {
            e.printStackTrace();
        } finally {
            if(reentrantLock.isHeldByCurrentThread()){
                reentrantLock.unlock();
            }
        }
    }
}

生产者消费者模式

在一个生产环境中，生产者和消费者在同一时间段内共享同一块缓冲区，生产者负责向缓冲区添加数据，消费者负责从缓冲区取出数据。

需要注意的是，当使用 lockObject.wait() 和 lockObject.notify时，wait()方法必须放在while(判断是否wait)的代码块中，如果用if的话会造成虚假唤醒。即多线程同时因为不满足条件而等待，当此锁对象被notify()时，同时唤醒，然后一个接一个抢夺资源执行命令，然而由于是if条件，所以当轮到后续线程执行命令时，不会因为不满足条件而再次wait.

容器类

package 多线程.WaitNotify;

public class Container {
    public Hamburger[] array = new Hamburger[6];
    public int index = 0;
    /**
     * 向容器中添加汉堡
     */
    public synchronized void push(Hamburger hamburger){
        //？？为什么是while，多线程wait时，notify唤醒相同对象进入wait的同步线程们，如果index最大后，又接一个push()就需要while来重复判断了
        //如果使用if 就是经典的虚假唤醒问题(可能被非notify的方法唤醒) wait一定要在while(条件)里使用
        //同资源wait碰到notify就会唤醒
        while(index == array.length){
            try {
                this.wait();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
        this.notify();
        array[index] = hamburger;
        index++;
        System.out.println("生产了一个汉堡"+hamburger);
    }
    /**
     * 从容器中取出汉堡
     */
    public synchronized Hamburger pop(){
        while(index == 0){
            //当前线程暂停
            //让正在访问当前资源的线程暂停
            try {
                this.wait();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
        //唤醒之前暂停的线程
        this.notify();
        index--;
        System.out.println("消费了一个汉堡"+array[index]);
        return array[index];
    }
}

有关wait、notify、notifyAll

三个方法是object的本地final方法，无法被重写，一般在sync同步代码块中使用这三个方法

wait：使当前线程阻塞，前提是要先获得锁，一般和synchronized配合使用（和lock会报错）。

​ 会释放当前锁，让出CPU，进入等待状态。

​ wait需要被try catch包围，以保证发生异常中断也可以使wait等待的线程唤醒。（？）

​ notiyf与wait要注意先后顺序

notify：唤醒当前线程。多个沉睡时，争夺到的变为运行态，其他继续等待。

notifyAll：唤醒所有沉睡的线程。这里指的是等待同一个锁对象的多个线程。

notify、notifyAll的区别：

锁池（等待获取sync的锁）

等待池（某线程调用了某对象的wait方法，调用wait后线程进入该对象的等待池）

notify随机唤醒（有竞争）对象等待池中一个线程进入锁池；notifyAll唤醒对象的等待池中所有线程，进入锁池。

wait()和notify()是用来 做线程同步用的， 他们都必须在synchronized语句块中调用。

进入synchronized块表示获得了对象锁，离开synchronized块表示释放了对象锁（对象锁释放了，其他等待此对象锁的线程就可以执行自己的synchronized语句块了）。

那么对象锁是个什么概念呢？

你可以把任何对象视做一把锁（? extends Object），锁是Java对象与生俱来的特性。

synchronized (mLock)， mLock就是一个对象（可以是任何对象），synchronized的意思就是尝试获得锁。如果得到了锁，那么其他线程的synchronized(mLock)语句将会阻塞，直到获得锁的synchronized语句块执行完毕或者mLock.wait()被调用。

当我们在synchronized块 中调用mLock.wait()方法，那么调用线程（线程A）就会阻塞在wait()语句那里，直到其他线程（线程B）调用mLock.notify()方法。

当mLock.notify()方法调用完成后，并不表示线程A可以立即执行，因为mLock.notify()方法也是在synchronized块中的，只有线程B继续执行直到离开synchronized块后（离开后就会释放mLock对象锁），线程A才能够获得对象锁并开始执行wait()之后的语句。

Lock下的等待与唤醒

使用 Lock 锁，就不能通过 wait 和 notify 来暂停线程和唤醒线程，而应该使用 Condition 的 await 和 signal 来暂停和唤醒线程。

package com.southwind.demo4;

import java.util.concurrent.locks.Condition;
import java.util.concurrent.locks.Lock;
import java.util.concurrent.locks.ReentrantLock;

public class Test {
    public static void main(String[] args) {
        Data data = new Data();
        new Thread(()->{
            for (int i = 0; i < 30; i++) {
                data.increment();
            }
        },"A").start();
        new Thread(()->{
            for (int i = 0; i < 30; i++) {
                data.decrement();
            }
        },"B").start();
    }
}

class Data{
    private Integer num = 0;

    private Lock lock = new ReentrantLock();

    private Condition condition = lock.newCondition();

    public void increment(){
        lock.lock();
        while(num!=0){
            try {
                condition.await();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
        num++;
        condition.signal();
        System.out.println(Thread.currentThread().getName()+"生产了汉堡"+num);
        lock.unlock();
    }

    public synchronized void decrement(){
        lock.lock();
        while(num == 0){
            try {
                condition.await();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
        num--;
        condition.signal();
        System.out.println(Thread.currentThread().getName()+"消费了汉堡"+num);
        lock.unlock();
    }
}

多线程并发卖票

一场球赛的球票分 3 个窗口出售，共 15 张票，用多线程并发来模拟 3 个窗口的售票情况

多线程可以使用双重检测，一个在锁外检测是否有执行资格，一个在锁内检测 当线程被阻塞后 竞争到资源后是否还满足执行资格。

package 多线程.多窗口卖票;

import java.util.concurrent.TimeUnit;
import java.util.concurrent.locks.ReentrantLock;

public class Ticket {
    private volatile int num = 20;
    private ReentrantLock lock = new ReentrantLock();

    public void sell(){

        while (num>=1){
            try {
                lock.lock();
                System.out.println(Thread.currentThread().getName()+"正在卖出...");
//                Thread.sleep(10);
                if (num>=1){
                    num--;
                    System.out.println("卖出成功，剩余 "+ num +" 张票" );
                } else {
                    System.out.println("已经卖光, 剩余"+num);
                }
                lock.unlock();
//                Thread.sleep(10);
            } catch (Exception e) {
                e.printStackTrace();
            }
        }
    }
}
package 多线程.多窗口卖票;

public class Test {
    public static void main(String[] args) {
        Ticket ticket = new Ticket();
        for (int i = 0; i < 3; i++) {
            Thread thread = new Thread(new Runnable() {
                @Override
                public void run() {
                    ticket.sell();
                }
            });
            thread.start();
        }
    }
}

ConcurrentModificationException

并发访问异常 常出现于读写同时操作

package com.southwind.demo5;

import java.util.ArrayList;
import java.util.List;
import java.util.concurrent.TimeUnit;

public class Test {
    public static void main(String[] args) {
        /*
            注意这里是ArrayList，本质是一个数组，所以多线程写时并没有报错
            因为多线程add时，即便对相同索引进行了编辑，也只是再覆盖一次
            但是一旦时LinkedList，本质变成了链表
            那么add时 尾指针指向错乱，从而导致后续的新元素空指针异常
        * */
        List<String> list = new ArrayList<>();
        for (int i = 0; i < 10; i++) {
            new Thread(()->{
                try {
                    TimeUnit.MILLISECONDS.sleep(1);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                //写
                list.add("a");
                //读
                System.out.println(list);
            },String.valueOf(i)).start();
        }
    }
}

如何解决？（修改add方法使其可以上锁）

1、Vector （只保证不出异常，并不能输出每次add后的list）

Vector的add方法（有同步）

public synchronized boolean add(E e) {
    modCount++;
    add(e, elementData, elementCount);
    return true;
}

ArrayLsit的add方法

public boolean add(E e) {
    modCount++;
    add(e, elementData, size);
    return true;
}

2、Collections.synchronizedList（List类型，且只保证不出异常，并不能输出每次add后的list）

private static void method1() {

    List list = Collections.synchronizedList(new ArrayList<>());

    for (int i = 0; i < 10; i++) {
        new Thread((()->{
            try {
                TimeUnit.MILLISECONDS.sleep(10);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            list.add("a");
            System.out.println(list);
        })).start();
    }

}

3、JUC：CopyOnWriteArrayList（保证不出异常，并不能输出每次add后的list）

CopyOnWrite 写时复制，当我们往一个容器添加元素的时候，不是直接给容器添加，而是先将当前容器复制一份，向新的容器中添加数据，添加完成之后，再将原容器的引用指向新的容器。

public boolean add(E e) {
    synchronized (lock) {
        Object[] es = getArray();
        int len = es.length;
        es = Arrays.copyOf(es, len + 1);
        es[len] = e;
        setArray(es);
        return true;
    }
}
