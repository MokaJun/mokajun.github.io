JUC 工具类

CountDownLatch：减法计数器

await()方法会阻塞当前线程，直至计数器满足条件会被自动唤醒

可以用来倒计时，当两个线程同时执行时，如果要确保一个线程优先执行，可以使用计数器，当计数器清零的时候，再让另一个线程执行。

package 多线程.JUC包;

import java.util.ArrayList;
import java.util.Collections;
import java.util.List;
import java.util.Vector;
import java.util.concurrent.CountDownLatch;

public class CountDownLatchTest {
    public static void main(String[] args) {
        //创建一个 CountDownLatch
        CountDownLatch countDownLatch = new CountDownLatch(11);

        new Thread(()->{
            for (int i = 0; i < 10; i++) {
                System.out.println("+++++++++++++++Thread");
                countDownLatch.countDown();
            }
        }).start();



        for (int i = 0; i < 10; i++) {
            System.out.println("try");
            try {
                //阻塞线程，等待条件达成自动唤醒
                countDownLatch.await();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            System.out.println("main-----------------");
        }

    }
}

coutDown()：计数器减一

await()：阻塞线程，等待条件达成自动唤醒，唤醒其他线程（可以多个，相当于闹钟 ）

new CountDownLatch(100)、coutDown()、await() 必须配合起来使用，创建对象的时候赋的值是多少，coutDown() 就必须至少执行多少次，否则计数器是没有清零的，计数器就不会停止，其他线程也无法唤醒，所以必须保证计数器清零，coutDown() 的调用次数必须大于构造函数的参数值。

若CountDownLatch赋初值为100，线程1循环200次执行coutDown，主线程等待清零后执行，那么在线程1循环100次后，就变成线程1与主线程交替执行。

CyclicBarrier：加法计数器

await()方法会阻塞当前线程，直至计数器满足条件会被自动唤醒

package 多线程.JUC包;

import java.util.concurrent.BrokenBarrierException;
import java.util.concurrent.CyclicBarrier;

public class CyclicBarrierTest {
    public static void main(String[] args) {
//        CyclicBarrier cyclicBarrier = new CyclicBarrier(4);
        CyclicBarrier cyclicBarrier = new CyclicBarrier(4,()->{
            System.out.println("放行");
        });

        for (int i = 0; i < 10; i++) {
            final int temp = i;
            new Thread(()->{
                //lambda表达式中不能访问非final外部栈变量。
                System.out.println("-->"+temp);
                try {
                    cyclicBarrier.await();
                } catch (InterruptedException e) {
                    e.printStackTrace();
                } catch (BrokenBarrierException e) {
                    e.printStackTrace();
                }
                System.out.println("<--");
            }).start();
        }
    }
}

await()：await()方法会阻塞当前线程，直至计数器满足条件会被自动唤醒，并且计数器是可以重复使用的，当计数器的线程执行完成一次之后，计数器自动清零，等待下一次执行。（每次调用await相当于对加法计数器进行一次加1）

new CyclicBarrier(30），for 执行 90 次，则计数器的任务会执行 3 次；假如for循环执行29次，后面0-29的线程就会阻塞。

Semaphore：计数信号量

实际开发中主要使用它来完成限流操作，限制可以访问某些资源的线程数量。

Semaphore 只有 3 个操作：

  • 初始化
  • 获取许可
  • 释放
public class SemaphoreTest {
    public static void main(String[] args) {
        //初始化
        Semaphore semaphore = new Semaphore(5);
        for (int i = 0; i < 15; i++) {
            new Thread(()->{
                //获得许可
                try {
                    semaphore.acquire();
                    System.out.println(Thread.currentThread().getName()+"进店购物");
                    TimeUnit.SECONDS.sleep(5);
                    System.out.println(Thread.currentThread().getName()+"出店");
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }finally {
                    //释放
                    semaphore.release();
                }

            },String.valueOf(i)).start();
        }
    }
}

每个线程在执行的时候，首先需要去获取信号量，只有获取到资源才可以执行，执行完毕之后需要释放资源（获取许可和释放是成对的操作，多个获取或多个释放经过测试都是不可行的，具体原因未知），留给下一个线程。

读写锁

先说一下为什么要用读写锁，展示一下错误案例

public class ConcurrentListAndMap {
    public static void main(String[] args) throws InterruptedException {
        /*
            tips:这里是ArrayList，本质是一个数组，所以多线程写时并没有报错
            因为多线程add时，即便对相同索引进行了编辑，也只是再覆盖一次
            但是一旦时LinkedList，本质变成了链表
            那么即便时只add而不读时 尾指针指向错乱，从而导致后续的新元素空指针异常
        * */
        List<String> list = new ArrayList<>();
        for(int i = 0; i < 10; i++){
            new Thread((()->{
                try {
                    TimeUnit.MILLISECONDS.sleep(10);
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
                /*
                    之所以ArrayList同时读写时会报错，是因为
                    读取集合打印时，调用了迭代器来逐个打印
                    在迭代打印过程中，里面有一个对比元素数量变化的方法，数量不对就抛出异常
                    因此我们对不满足的集合类型需要使用 读写锁 ，或者直接改变当前集合类型为其他集合
                * */
                list.add("a");
                System.out.println(list);
            })).start();
        }
    }

接口 ReadWriteLock，实现类是 ReentrantReadWriteLock，可以多线程同时读，但是同一时间内只能有一个线程进行写入操作。

读写锁也是为了实现线程同步，只不过粒度更细，可以分别给读和写的操作设置不同的锁。

package 多线程.JUC包;

import java.util.ArrayList;
import java.util.HashMap;
import java.util.Map;
import java.util.Objects;
import java.util.concurrent.locks.ReadWriteLock;
import java.util.concurrent.locks.ReentrantReadWriteLock;

public class ReadWriteLockTest {
    public static void main(String[] args) {
        /*
            tab[i]相同索引处本质是链表形式，同时编写就会导致尾指针错乱，出现覆盖问题
            上了写锁之后，在一个线程写入过程中，禁止其他线程写与读
            上了读锁之后，在一个线程读取过程中，其他线程也可以读，但是不能写
        * */
        Cache cache = new Cache();
        for (int i = 0; i < 5; i++) {
            final int temp = i;
            new Thread(new Runnable() {
                @Override
                public void run() {
                    try {
                        Thread.sleep(100);
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                    cache.write(temp,String.valueOf(temp));
                }
            }).start();
            new Thread(new Runnable() {
                @Override
                public void run() {
                    try {
                        Thread.sleep(100);
                    } catch (InterruptedException e) {
                        e.printStackTrace();
                    }
                    cache.read(temp);

                }
            }).start();
        }

    }
}

class Cache{
    private Map<Integer,String> map = new HashMap<>();
    private ArrayList<String> arr = new ArrayList<>();
    private ReadWriteLock readWriteLock = new ReentrantReadWriteLock();

    /**
     * 写操作
     */
    public void write(Integer key,String value){
        readWriteLock.writeLock().lock();
        System.out.println("    "+key+"开始写入");
        map.put(key,value);
        System.out.println("    "+key+"写入完毕");
        readWriteLock.writeLock().unlock();
    }

    /**
     * 读操作
     */
    public void read(Integer key){
        readWriteLock.readLock().lock();
        System.out.println(key+"开始读取");
        map.get(key);
        System.out.println(key+"读取完毕");
        readWriteLock.readLock().unlock();
    }
}

写入锁也叫独占锁，只能被一个线程占用，读取锁也叫共享锁，多个线程可以同时占用。（读取锁应该主要保证在数据未写入时的阻塞，写操作开始的时候，所有晚于写操作的读操作都会进入等待状态，只有写操作完成并通知后才会将等待的线程唤醒继续执行）
