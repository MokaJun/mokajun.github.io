Tomcat入门

  1. Tomcat是一个服务，部署于服务器上

    其中的webapp文件夹装载的就是用来给客户端访问的java程序（需要被打包成war）

    我们IDEA集成Tomcat其实就是每次运行自动将java程序打包成war，然后丢进Tomcat/webapp

  1. Tomcat目录。

    bin：存放各个平台下启动和停⽌ Tomcat 服务的脚本⽂件。

    conf：存放各种 Tomcat 服务器的配置⽂件。 其中server.xml配置Tomcat

    lib：存放 Tomcat 服务器所需要的 jar。

    logs：存放 Tomcar 服务运⾏的⽇志。

    temp：Tomcat 运⾏时的临时⽂件。

    webapps：存放允许客户端访问的资源（Java 程序）。

    work：存放 Tomcat 将 JSP 转换之后的 Servlet ⽂件。

  2. 更改浏览器默认地址

    run-> edit config

    在Application context处设置浏览器访问地址的对应名称，config页面url是根据这个改变的

    Application context保留/即可

新建Tomcat项目

首先必须是旗舰版IDEA才可以创建Web项目

新版IDEA中 没有javaEE，取而代之的是在左侧列表中，右键单击 选择 add frame来添加

随后外部libraries也需要手动添加，在project structure -> modules -> dependence里的 + library

Servlet使用

一如server.xml是配置Tomcat的

web.xml在IDEA中就是配置servlet的

想让浏览器访问servlet

使用Servlet就是实现Servlet接口
但是浏览器无法访问WEB-INF文件，通过映射方式实现：
    1.在web.xml文件里设置servlet的名字-全包名 设置servlet-mapping的名字-浏览器地址名
        <servlet>
            <servlet-name>servletName</servlet-name>
            <servlet-class>Myservlet</servlet-class>
        </servlet>
        <servlet-mapping>
            <servlet-name>servletName</servlet-name>
            <url-pattern>/address</url-pattern>
        </servlet-mapping>
    2.注解

浏览器网址request获取参数：

网址：servletName?key=value

然后通过Servlet类中的servletRequest.getParameter(“key”);获取同名参数的value

String value = servletRequest.getParameter("id");
System.out.println("我是Servlet，我已经接收到了客户端发来的请求，参数是"+value);

Response回复结果：

servletResponse.setContentType("text/html;charset=UTF-8");
servletResponse.getWriter().write(value);
